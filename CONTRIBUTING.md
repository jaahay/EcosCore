# Contributing to Core Event & State System

Thank you for your interest in contributing! We welcome contributions that improve the quality, clarity, and functionality of this project.

## How to Contribute

### Reporting Issues

- Please open issues for bugs, feature requests, or documentation suggestions.
- Provide clear descriptions and steps to reproduce bugs.

### Submitting Pull Requests

- Fork the repository and create a feature branch.
- Write clear, concise commit messages.
- Follow the existing code style and conventions.
- Include unit tests for new features or bug fixes.
- Ensure all tests pass before submitting.
- Describe your changes thoroughly in the PR description.

### Code Style

- Follow consistent naming, formatting, and documentation style as seen in the codebase.
- Use C++20 features thoughtfully, maintaining readability and performance.

### Testing

- Add or update unit tests in `/tests/core/`.
- Run all tests locally before submitting.

### Communication

- Engage respectfully and constructively in discussions.
- Ask questions if unsure about design or implementation.
